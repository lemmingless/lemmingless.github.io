fast map match环境配置问题

总体按照Fast map matching配置，其中安装过程删去了anaconda环境

1 问题

wget https://raw.githubusercontent.com/transcode-open/apt-cyg/master/apt-cyg 这一步

1.1 错误：wget: unable to resolve host address ‘raw.githubusercontent.com’

检查DNS设置

1.1.1 打开网络连接设置：

在任务栏搜索框中输入“网络连接设置”并打开。
或者，您可以在控制面板中找到“网络和共享中心”，然后点击“更改适配器设置”。

1.1.2 选择网络适配器：

找到您当前连接的网络适配器（通常是无线或以太网），右键点击并选择“属性”。

1.1.3 选择IPv4协议属性：

在属性窗口中找到“Internet 协议版本 4 (TCP/IPv4)”并双击打开属性窗口。

1.1.4 手动配置DNS服务器：

在IPv4属性窗口中，选择“使用以下 DNS 服务器地址”。
输入首选和备用DNS服务器的地址。您可以使用公共DNS服务器，如Google DNS（8.8.8.8和8.8.4.4）或Cloudflare DNS（1.1.1.1和1.0.0.1）。
另一种方法见https://www.cnblogs.com/feipeng8848/p/8555648.html

2 问题

apt-cyg install make gcc-g++ cmake gdal libboost-devel libgdal-devel libexpat1-devel libbz2-devel zlib-devel swig python-devel 这一步

2.1 错误：Unable to locate package libexpat1-devel

改为装libexpat-devel

3 问题

git clone git@github.com:cyang-kth/fmm.git . 这一步

3.1 错误：fatal: Could not read from remote repository.

使用git bash在\cygwin64\home\13532\fmm文件夹下git clone git@github.com:cyang-kth/fmm.git .

4 问题

cmake .. 这一步

4.1 错误：没有cyguv-1.dll

重装执行安装cygwin的流程，如下图所示步骤中搜索libuv相关的未安装的依赖，全部安装

4.2 错误：Could NOT find SWIG (missing: SWIG_EXECUTABLE SWIG_DIR)

同上图，安装python3-devel

5 问题

make -j8 这一步
将gdal和libgdal-devel版本从3.8.3退回到了3.6.2，同时安装了libgdal32、libgdal34、python-gdal、python3-gdal

6 结语

流程走下来安装完成，没有报错，具体使用暂未尝试，可能仍存在问题
